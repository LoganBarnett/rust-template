* TODO Darwin services must log to /var/log/<service>/ with rotation
All darwin launchd services should write ~StandardOutPath~ and
~StandardErrorPath~ to ~/var/log/<service>/~ rather than ~/tmp/~ or
nowhere.  ~/tmp/~ is cleared on reboot and is not where service logs
belong.  Without explicit log paths, launchd silently discards all
output — making post-mortem diagnosis impossible.

** Log directory creation

The log directory must exist before launchd starts the agent.  Use an
activation script in the module's ~config~ block:

#+begin_src nix
system.activationScripts.postActivation.text = ''
  mkdir --parents /var/log/<service>
  chmod 755 /var/log/<service>
'';
#+end_src

While not included here, you will also need to ~chown~ the directory if this is
a user service.

** launchd plist

Set both ~StandardOutPath~ and ~StandardErrorPath~ in the agent's
~serviceConfig~:

#+begin_src nix
StandardOutPath = "/var/log/<service>/<service>.log";
StandardErrorPath = "/var/log/<service>/<service>.log";
#+end_src

** Log rotation via newsyslog

macOS ships ~newsyslog~ which reads drop-in configs from
~/etc/newsyslog.d/~.  Add a config via ~environment.etc~:

#+begin_src nix
environment.etc."newsyslog.d/<service>.conf".text = ''
  # logfilename                          mode count size  when flags
  /var/log/<service>/<service>.log        644  5     10240 *    NJ
'';
#+end_src

- ~N~ = no signal (the service isn't syslogd; it re-opens the file on
  next write).
- ~J~ = bzip2 compression for rotated archives.
- ~size~ is in KB (10240 = 10 MB).
- ~count~ is the number of rotated archives to keep.

Adjust ~count~ and ~size~ per service — high-volume services (e.g. Ollama
inference server) need more generous limits than one-shot helpers.

** Reference commit

See =dotfiles= commit =a223015e= (~fix(ollama): add launchd log output
with newsyslog rotation~) for a working example applied to the Ollama
darwin module.

* TODO MergeConfig derive macro
Auto-generate the ~CliRaw -> ConfigFileRaw -> Config~ merge from a single type
definition.  The ~#[derive(MergeConfig)]~ macro should produce the ~CliRaw~
(clap Parser), ~ConfigFileRaw~ (serde Deserialize), ~ConfigError~, the
~from_cli_and_file~ resolution function, and the ~CliApp~ trait impl from a
single annotated ~Config~ struct.

** API design

Struct attributes:
- ~app_name = "name"~ — required, used for ~find_config_file~ and ~CliApp::app_name~
- ~extra_cli = "Type"~ — optional type flattened into generated ~CliRaw~
- ~extra_file = "Type"~ — optional type flattened into generated ~ConfigFileRaw~

Field attributes:
- ~common~ — resolved from ~CommonCli~ / ~CommonConfigFile~ via ~resolve_log_settings~
- ~env = "VAR"~ — environment variable for clap
- ~name = "field"~ — CLI/file field name (defaults to struct field name)
- ~short = 'x'~ — short flag for clap
- ~default = "expr"~ — default value expression
- ~required~ — error if not provided (no default)
- ~parse~ — CLI/file type is ~String~, parsed via ~FromStr~ to field type
- ~skip~ — not auto-merged; calls ~Config::resolve_<field>(cli, file)~
- ~cli_only~ — not in config file

Generated items:
1. ~CliRaw~ struct (~#[derive(Debug, clap::Parser)]~)
2. ~ConfigFileRaw~ struct (~#[derive(Debug, serde::Deserialize, Default)]~)
3. ~ConfigError~ enum (~File~ + ~Validation~ variants)
4. ~impl Config { pub fn from_cli_and_file(cli: CliRaw) -> Result<Self, ConfigError> }~
5. ~impl CliApp for Config~

Example (simple CLI):
#+begin_src rust :exports code
#[derive(Debug, Clone, MergeConfig)]
#[merge_config(app_name = "my-cli")]
pub struct Config {
    #[merge_config(common)]
    pub log_level: LogLevel,
    #[merge_config(common)]
    pub log_format: LogFormat,
    #[merge_config(short = 'n', default = "\"World\".to_string()")]
    pub name: String,
}
#+end_src

Example (server with OIDC):
#+begin_src rust :exports code
#[derive(Debug, Clone, MergeConfig)]
#[merge_config(
    app_name = "my-server",
    extra_cli = "OidcCliFields",
    extra_file = "OidcFileFields",
)]
pub struct Config {
    #[merge_config(common)]
    pub log_level: LogLevel,
    #[merge_config(common)]
    pub log_format: LogFormat,
    #[merge_config(name = "listen", env = "LISTEN",
                   default = "\"127.0.0.1:3000\".to_string()", parse)]
    pub listen_address: ListenerAddress,
    #[merge_config(env = "FRONTEND_PATH",
                   default = "std::path::PathBuf::from(\"frontend/public\")")]
    pub frontend_path: std::path::PathBuf,
    #[merge_config(env = "BASE_URL", required)]
    pub base_url: String,
    #[merge_config(skip)]
    pub oidc: Option<OidcConfig>,
}
#+end_src
