#+title:     Contributing to rust-template
#+language:  en

Guidelines for contributing to the template repository itself.  Derived
projects have their own =CONTRIBUTING.org= with project-specific conventions.

* Document conventions

- *Root-level UPPER_CASE names* are reserved for the standard set: =README=,
  =CONTRIBUTING=, =LICENSE=.  All other documentation lives in =docs/= with
  lower-kebab-case file names (e.g. =docs/getting-started.org=).
- Every document must be reachable — directly or indirectly — from =README.org=.
- Use org-mode for all new documents.

* Development environment

Enter the Nix dev shell before running any commands:

#+begin_src sh :results none :exports code
nix develop
#+end_src

Or, if you use direnv:

#+begin_src sh :results none :exports code
direnv allow
#+end_src

* Formatting

Run =cargo fmt= before committing.  The pre-commit hook enforces this
automatically, so commits made outside the Nix shell may be rejected if
=rustfmt= is not on =PATH=.

* Testing

Run the full test suite after every change:

#+begin_src sh :results none :exports code
nix develop --command cargo test --workspace
#+end_src

Tests must exit with a proper error code.

* UI philosophy: no modals

Do not use modals anywhere in the UI.  Replace every modal with an
inline expansion that keeps surrounding context visible.

** Why

Modals block the UI — which may contain information relevant to the
decision at hand — and they are an accessibility nightmare (backdrop
divs, tab-index trapping, z-index layering).  As an interface matures,
the probability of needing a modal inside a modal trends to 1.  Powerful
UIs do not use modals.

** Patterns

- *Export / large output*: clicking the trigger button slides the
  surrounding content down and reveals the output inline, in the normal
  document flow.
- *Destructive confirmations*: replace the triggering button in-place
  with a "Confirm delete?  Yes / No" prompt.  When the user decides (or
  clicks elsewhere), restore the original button.
- *Progressive disclosure*: expand a section in-place rather than
  overlaying.

Every piece of revealed UI must live in the normal document flow.  No
backdrop divs, no tab-index trapping, no z-index layering.

* Commit workflow

1. Make your changes on a feature branch.
2. Ensure =cargo fmt= and =cargo clippy --workspace= produce no warnings.
3. Run =cargo test --workspace= and confirm all tests pass.
4. Open a pull request against =main=.
